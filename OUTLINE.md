# 全书大纲 — 从零构建校园问答助手：RAG全栈实战开发指南

## 书籍元数据

- **书名**：从零构建校园问答助手：RAG全栈实战开发指南
- **作者**：llmlearning-x
- **目标读者**：在校大学生 / 实训学员（零基础到入门）
- **风格**：O'Reilly 实战派 — 概念 → 代码 → 最佳实践 → 常见坑
- **版本**：v1.0（20260408）
- **语言**：中文正文，代码/术语保留英文原文

## 全书结构

本书以「6天实训项目」为主线，每章对应一天的开发任务，带领读者从零开始搭建一个
完整的校园知识问答系统（RAG + LLM + 全栈）。

## 第4章：知识库与向量化处理（Day 4 — 业务模块开发）

**核心问题**：「怎样让系统「记住」上传的文档？」

### 4.1 知识库模块 CRUD
- 文档上传（multipart/form-data）
- 文档列表与搜索
- 文档更新与重新处理
- 文档删除（级联清理向量数据）

### 4.2 文档处理：文本切分（Chunking）
- 为什么要切分文档？
- 固定长度切分 vs 语义切分
- 重叠（overlap）的作用
- 实战：用 LangChain TextSplitter

### 4.3 向量化处理（Embedding）
- Embedding 模型选型（OpenAI / 本地模型）
- 批量 Embedding 接口调用
- 向量维度与存储格式

### 4.4 向量库构建（FAISS）
- FAISS 基础：IndexFlatL2 vs IndexIVFFlat
- 向量持久化：保存与加载
- 增量更新策略

### 4.5 后台管理：知识库管理页面
- 文件上传进度展示
- 处理状态轮询
- 知识库统计面板

### 4.6 前端：文档上传与管理界面
- Element Plus Upload 组件
- 文档列表与操作按钮
- 删除确认对话框

**本章输出**：代码提交、日报、知识库数据（向量文件）、知识库 CRUD 模块
