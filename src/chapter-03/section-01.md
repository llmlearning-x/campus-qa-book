# 3.1 业务流程与系统流程图

## 为什么先画流程图？

流程图不是形式主义。在开始写代码之前，用流程图把业务逻辑捋清楚，可以：
- 发现业务边界不清晰的地方（提前沟通，而不是写完再改）
- 让前后端对整体流程达成共识
- 作为开发文档，方便后续维护

---

## 系统总体流程图

```
用户进入系统
    │
    ├──► 未登录？──► 跳转登录页
    │                  │
    │              登录/注册
    │                  │
    └──► 已登录 ◄───────┘
            │
            ├──► 普通用户 ──► 问答聊天页面
            │                    │
            │               提交问题
            │                    │
            │             RAG 检索 + LLM 生成
            │                    │
            │               返回答案
            │
            └──► 管理员 ──► 知识库管理
                                │
                           上传文档
                                │
                      文档处理（切分+向量化）
                                │
                           知识库就绪
```

---

## RAG 流程图

```
【离线：建立知识库】
管理员上传文档（PDF/TXT/DOCX）
    │
    ▼
提取文本内容
    │
    ▼
文本切分（每段500字，重叠50字）
    │
    ▼
批量 Embedding（调用 OpenAI API）
    │
    ▼
向量 + 原文 存入 FAISS
    │
    ▼
更新 MySQL documents.status = 'done'


【在线：回答问题】
用户输入问题
    │
    ▼
问题 Embedding
    │
    ▼
FAISS 检索 Top-5 相似片段
    │
    ▼
构建 Prompt（系统指令 + 检索片段 + 用户问题）
    │
    ▼
调用 LLM API（流式）
    │
    ▼
SSE 实时推送给前端
    │
    ▼
保存问答记录到 MySQL
```

## 绘图工具推荐

| 工具 | 特点 | 适合场景 |
|------|------|---------|
| draw.io（diagrams.net） | 免费、功能全、离线可用 | 系统架构图、流程图 |
| Figma | 协作强，适合团队 | UI 设计、界面原型 |
| Excalidraw | 手绘风格，快速草图 | 头脑风暴、快速讨论 |
| PlantUML | 代码生成图，版本友好 | 时序图、类图 |

**推荐 Excalidraw** 用于手绘风格的快速草图，直接访问 [excalidraw.com](https://excalidraw.com) 即可使用。

---

## 本节小结

- 流程图是沟通工具，不是文档形式
- 系统流程图展示用户视角的操作路径
- RAG 流程图展示技术层面的数据流转
- 绘图工具选自己顺手的即可
