# 第2章：工程基础搭建

> **核心问题**：怎样让一个项目从第一天就「跑起来」？

---

「先让它跑起来，再让它跑得好。」

这是软件开发中一条重要的工程原则。很多初学者会陷入「完美主义陷阱」——在真正写功能之前，想把每个细节都设计得无懈可击。结果往往是什么都没做出来。

Day 2 的目标就是：**建立工程基础，让一个最小版本的系统真正跑起来**。

具体来说：
- 数据库能连上
- 后端 API 能返回数据
- 前端能调用后端
- 代码能推送到 GitLab

这些看起来很基础，但它们是后续所有功能开发的基石。
