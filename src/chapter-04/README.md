# 第4章：知识库与向量化处理

> **核心问题**：怎样让系统「记住」上传的文档？

---

今天是本书技术含量最高的一章之一。

我们要实现的是 RAG 系统的「记忆」部分：

1. **上传文档** → 保存到服务器
2. **提取文本** → 读懂文档内容
3. **切分文本** → 把长文档切成合适的小块
4. **向量化** → 把每个小块变成数字向量
5. **存入向量库** → 建立可检索的语义索引

这个过程完成后，系统就有了「记忆」——它知道文档里写了什么，并能快速找到与任意问题最相关的片段。

### 知识库构建流程

```
上传文档 ──► 文本加载 ──► 文本切分 ──► 向量化 ──► 存入FAISS
```
