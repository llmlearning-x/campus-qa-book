# 5.3 Prompt 工程

## 为什么 Prompt 很重要？

同样的大模型，不同的 Prompt 会得到截然不同的结果。

**坏 Prompt**：
```
你是一个助手，回答用户的问题。
```

**好 Prompt**（我们使用的）：
```
你是一个校园知识问答助手，专门帮助同学解答与学校相关的问题。

## 回答原则
1. 基于资料回答：只根据下方提供的参考资料作答
2. 诚实面对未知：如果资料中没有相关信息，明确说「无法回答」
3. 简洁准确：回答简洁清晰，避免废话

## 参考资料
{context}
```

差别在于：
1. **角色设定**：告诉模型它是谁，它会更认真地扮演这个角色
2. **行为约束**：明确说「只基于资料回答」，防止模型编造信息（幻觉）
3. **结构化输出**：Markdown 格式让指令更清晰

---

## 防幻觉技巧

LLM 最大的问题是「幻觉」（Hallucination）：模型会自信地编造不存在的事实。

**防幻觉的 Prompt 技巧**：

1. **显式约束**：「只根据以下资料回答」
2. **提供退路**：「如果资料中没有，请说不知道」
3. **要求引用**：「请注明信息来自哪条资料」

```python
SYSTEM_PROMPT = """...
如果参考资料中没有足够信息回答问题，请明确回答：
「根据现有资料，我无法找到关于这个问题的准确信息。建议您直接咨询学校相关部门。」

不要基于常识或猜测回答，只使用提供的参考资料。
..."""
```

## 温度（Temperature）参数

`temperature` 控制模型输出的随机性：

| temperature | 效果 | 适用场景 |
|------------|------|---------|
| 0.0 | 确定性输出，每次相同 | 需要精确答案（如数学计算） |
| 0.3~0.7 | 适度灵活，推荐 | **问答助手（我们用 0.7）** |
| 1.0+ | 高度随机，创意丰富 | 写作、头脑风暴 |

对于知识问答，0.7 是个不错的平衡点：既有一定的表达灵活性，又不会随机到乱说。

---

## 本节小结

- 明确的角色设定和行为约束是好 Prompt 的基础
- 「只基于资料回答 + 无法回答时诚实说」是防幻觉的核心策略
- temperature=0.7 适合问答场景
