# 6.5 项目成果展示与讲解

## 演示流程（建议 10 分钟）

一个好的 Demo 演示需要提前规划，不能临场发挥。

### 推荐演示脚本

```
[1分钟] 项目背景介绍
  - 痛点：学生找不到学校信息，经常要问辅导员
  - 方案：AI 问答助手，上传文档即可自动回答

[2分钟] 技术架构介绍
  - 展示架构图：前端 → 后端 → MySQL + FAISS → LLM
  - 重点介绍 RAG 原理（用类比：先找资料，再回答）

[5分钟] 现场演示
  1. 管理员登录
  2. 上传一份学校文档（提前准备好，确保处理完成）
  3. 切换到聊天页面
  4. 提问 3~5 个问题（提前测试过，确保能回答）
  5. 展示问答历史记录

[2分钟] Q&A
  准备好常见问题的回答（见下方）
```

### 常见问题准备

**Q: AI 会不会回答错误？**
> 我们的系统只基于上传的文档回答，如果文档里没有的信息，AI 会明确说找不到相关资料。相比直接问 ChatGPT，我们的系统不会「胡说」。

**Q: 系统安全吗？**
> 我们使用 JWT Token 认证，密码用 bcrypt 哈希存储，文件上传有类型和大小限制。

**Q: 能支持多少文档？**
> FAISS 支持百万级向量检索，对于学校文档规模（几百份）完全够用。

## PPT 结构建议

1. 封面（项目名 + 团队成员）
2. 项目背景与价值
3. 技术架构图
4. 核心功能演示截图
5. 技术挑战与解决方案
6. 项目成果（代码量、功能数量、测试通过率）
7. 后续改进方向
8. 致谢
