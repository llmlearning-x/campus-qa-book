# 6.2 系统联调测试

## 测试清单

联调测试不是随便点几下，而是系统性地走完所有关键流程：

### 🔐 用户认证流程
- [ ] 注册新用户（用户名重复时提示错误）
- [ ] 登录成功，获取 Token
- [ ] 登录失败（密码错误，提示正确错误信息）
- [ ] Token 过期后自动跳转登录页
- [ ] 退出登录清除本地状态

### 📚 知识库管理流程
- [ ] 上传 PDF 文档，状态从 pending → processing → done
- [ ] 上传不支持的文件类型（如 .jpg），提示错误
- [ ] 上传超大文件（> 50MB），提示文件太大
- [ ] 文档列表正确展示状态和切片数量
- [ ] 删除文档，确认向量库同步清除

### 💬 问答流程
- [ ] 上传文档后，提问相关问题，能得到准确回答
- [ ] 流式输出正常，打字机效果流畅
- [ ] 提问知识库中没有的问题，AI 诚实回答「无法找到相关信息」
- [ ] 问答历史记录正确保存并可查询

### 👥 用户管理（管理员）
- [ ] 管理员可以查看用户列表
- [ ] 禁用用户后，该用户登录被拒绝
- [ ] 普通用户访问管理员页面，被重定向

## 用 Postman 进行接口测试

对于关键接口，建议用 Postman 直接测试，排除前端干扰：

```
POST http://localhost:8000/api/users/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}

期望响应：
{
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiJ9...",
    "token_type": "bearer",
    "user": {"id": 1, "username": "admin", "role": "admin"}
  }
}
```

## Bug 分类与优先级

发现 Bug 时按优先级处理：

| 优先级 | 定义 | 处理策略 |
|--------|------|---------|
| P0（阻塞）| 核心流程无法运行 | 必须今天修复 |
| P1（严重）| 功能有问题但有绕过方式 | 尽力修复 |
| P2（一般）| UI 细节问题 | 时间允许修复 |
| P3（建议）| 优化建议 | 记录到后续迭代 |

---

## 本节小结

- 联调测试要系统性覆盖所有关键流程，不能只测「正常路径」
- 异常路径测试（错误输入、越权访问）同样重要
- Bug 分级处理，确保 P0 问题在发布前修复
