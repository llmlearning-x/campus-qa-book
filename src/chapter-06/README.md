# 第6章：测试、发布与项目总结

> **核心问题**：一个项目怎样才算「完成」？

---

今天是实训的最后一天。

经过5天的奋战，系统的所有功能模块都已开发完毕。但一个项目能「跑」和能「交付」之间，还有一段距离。

**今天我们要做的事**：
- 联调所有模块，修复集成 Bug
- 打磨 UI 细节（加载状态、错误提示、响应式布局）
- 打包并部署到服务器
- 准备演示，做好项目总结

## 本章学习目标

- [ ] 完成系统联调测试，关键功能无阻塞性 Bug
- [ ] 前端完成生产构建（`npm run build`）
- [ ] 系统成功部署，通过 IP 可访问
- [ ] 项目 PPT 和演示脚本准备好

## 本章产出

- 代码提交（final tag）
- 编译包（前端 dist/）
- 日报
- 项目总结文档
- 人员评估表
